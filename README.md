# Finance Dashboard Backend

A role-based backend for a finance dashboard providing User Management, Financial Record CRUD and Dashboard Aggregations.

## Features
- **Role-based Access Control:** Viewer, Analyst, and Admin roles natively supported.
- **Financial Record Management:** CRUD and filtering by type/category.
- **Dashboard Summaries:** Aggregated totals and category-level breakdowns.
- **Database:** SQLite initialized by default, but completely parameterized for PostgreSQL.
- **Authentication:** JWT based auth wrapper.

## Setup Instructions

1. **Environment Setup**
   ```bash
   cp .env.example .env
   ```

2. **Docker Setup (Recommended)**
   ```bash
   docker-compose up --build
   ```

3. **Local Setup**
   ```bash
   python -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   
   alembic upgrade head
   python app/db/init_db.py
   uvicorn app.main:app --reload
   ```

## API Documentation

Swagger API documentation is automatically generated by FastAPI:
- Wait for the app to start
- Go to `http://localhost:8000/docs`

### Default Admin Credentials
When you run `init_db.py` (which runs automatically in docker container), a default admin user is seeded:
- **Email:** `admin@finance.local`
- **Password:** `adminpassword`

## Assumptions & Simplifications
- **Soft Deletion:** Fields to implement soft delete are commented within the schema. Could be implemented as an addition.
- **Database Engine:** Opted for SQLite inside Docker/local for immediate out-of-the-box functioning without extra config.
